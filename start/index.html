The template generates a complete example project w/ corresponding API for you!

Let's look at the User resource.

Click on the User resource under the Resources section.

This is our User resource for this project.

Click the Edit tab.

Here we can edit the Title, Name, and Path of our Resource. We can also add, edit, or remove fields in our Resource.

Click the settings button for the Email field.

Here are the display settings for the Email field.

We configured the Label and Placeholder of this field, and require its values to be Unique.

Click the Validation tab.

Here are the validation settings for the Email field.

We configured this field to be Required.

Click the API tab.

Here we can configure the Property Name used for this field in API calls.

This is usually automatically generated for you, so no need to worry about it.

Feel free to check out the settings of the other fields!

Next let's look at the User Login Form.

Click on the Edit button for User Login Form on your project home page.

This form is embedded with the User resource fields needed to log in.

Click on the Permissions tab.

Giving the Anonymous Role the Create Own Submissions Permission lets anonymous users submit this login form.

Click on the Actions tab.

Here are the Actions for our Form. Actions are performed when a Form is submitted.

Click on the Authentication Action.

This action uses the Form submission data to authenticate our Users.

The User Registration Form is similar to the Login Form, with a few differences.

Click on on the Edit button for User Registration Form on your project home page.

The User Register Form also has a User Full Name and a Verify Password field.

Click on the settings button for the Verify Password field.

Click on the Validation tab, then click on Custom Validation.

This field validates if the passwords match by using custom validation logic.

Click on the Permissions tab.

Giving the Anonymous Role the Create Own Submissions Permission lets anonymous users submit this registration form.

Click on the Actions tab.

Click on the Authentication Action.

For this registration form, we authenticate New Resources.

Finally, lets look at the Director and Movie Resources.

Click on Director under the Resources section.

The Director Resource contains Name, Phone Number, and Address fields.

Click on the Permissions tab.

These permissions allow Authenticated Users to manage their own Directors and see other Users' Directors, while only Administrator Users can update or delete other Directors.

The Movie Resource is similar to Director but has the Title, Video, and Director fields.

Handling I/O with the Formio factory

app/scripts/app.js
.state('createMovie', {
    url: '/create/movie',
    templateUrl: 'views/movie/create.html',
    controller: ['$scope', '$state', function ($scope, $state) {
        $scope.$on('formSubmission', function(event, submission) {
            $state.go('movie.view', {movieId: submission._id});
        });
    }]
})
.state('movie.view', {
    url: '/',
    parent: 'movie',
    templateUrl: 'views/movie/view.html',
    controller: ['$scope', '$stateParams', 'Formio', function ($scope, $stateParams, Formio) {
        var loader = new Formio($scope.movieForm + '/submission/' + $stateParams.movieId);
        $scope.movie = {};
        loader.loadSubmission().then(function(submission) {
            $scope.movie = submission;
        });
    }]
})
